You are extracting the logical decomposition of one clinical trial eligibility item.

Task:
Given ONE top-level eligibility item, split it into the smallest meaningful atomic clauses while preserving the original logical meaning.

Important:
- The input is ONE top-level item only, not a whole multi-line section.
- This is Pass 1 only.
- Do NOT infer logical connectors across unrelated top-level items.
- Do NOT extract entity_type, operator, value, unit, temporal_context, normalized_concept, computability, provenance, or history_context.
- criterion_type must follow the Criterion Type Hint from the input.
- evidence_text must be copied verbatim from the source text.
- connector_to_next describes how this clause connects to the NEXT clause in the list.
- For the LAST clause in the list, connector_to_next MUST be null.
- Use quantifier only when the text explicitly expresses cardinality, such as:
  - "at least two of the following"
  - "at most one"
  - "exactly three"
- Use is_negated=true for explicitly negated clauses such as "no prior treatment..." or "without evidence of..."
- connector_to_next should represent the logical relationship between the resulting atomic clauses after negation has been applied.
- When a negation word such as "no", "neither", or "without" scopes over multiple conditions joined by "or", set is_negated=true for each affected clause and use connector_to_next="AND" between the negated clauses. This represents NOT A AND NOT B. Do not copy the surface "or" as the connector in this case.
- Given ONE top-level eligibility item, split it into the smallest clinically complete atomic clauses while preserving the original logical meaning.
- Do not normalize concepts, invent medical facts, or infer information not present in the source text.
- If the sentence contains a difficult exception that cannot be safely decomposed without guessing, keep the clause shallow and preserve only directly segmentable logic.
- evidence_text must be an exact verbatim substring from the source text. It may be shared across clauses when the source uses coordination, shared negation, a quantifier list, or a shared clinical head noun. The selected span should support the full clause_text.
- If safe decomposition would require inventing missing words or losing the shared clinical head noun, keep the phrase as one shallow clause.

Return ONE valid JSON object with exactly this structure:

{
  "trial_id": "string",
  "criterion_type": "inclusion" | "exclusion",
  "clauses": [
    {
      "clause_text": "atomic clause text",
      "evidence_text": "exact substring from source text",
      "is_negated": boolean,
      "connector_to_next": "AND" | "OR" | null,
      "quantifier": null or {
        "quantifier_type": "AT_LEAST" | "AT_MOST" | "EXACTLY",
        "value": integer
      }
    }
  ]
}
